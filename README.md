# Кейс-задача №3. Разработка web-приложения для блога

## 1. Описание проекта

В рамках выполнения кейс-задачи было разработано web-приложение для ведения блога с возможностью регистрации пользователей, публикации постов, подписок, комментирования и управления контентом. Основная цель проекта — создание удобной платформы для взаимодействия пользователей через публикацию текстовых материалов.

Приложение реализовано как многофункциональная система с разграничением доступа и возможностью персонализации контента.



Анализ выполненной задачи

## 1. Реализация системы авторизации

Была успешно создана система регистрации и входа пользователей. Основной сложностью являлась организация безопасного хранения паролей и проверки данных.

---

## 2. Организация базы данных

Для хранения информации была спроектирована структура базы данных, включающая таблицы:

* users;
* posts;
* comments;
* subscriptions;
* tags.

Это позволило организовать логические связи между сущностями.

---

## 3. Реализация персональной ленты

Наиболее важной частью проекта стала генерация ленты публикаций на основе подписок. Для этого использовались запросы к базе данных с фильтрацией по подпискам пользователя.

---

## 4. Разграничение прав доступа

Была реализована система проверки прав:

* только автор может редактировать пост;
* удаление доступно владельцу;
* скрытые посты доступны ограниченному кругу лиц.

Это повысило безопасность системы.

---

## 5. Работа с тегами

Система тегов позволила улучшить навигацию и упростить поиск информации среди большого количества публикаций.

---

## 6. Реализация комментариев

Комментарии обеспечили взаимодействие между пользователями и повысили социальную активность внутри платформы.

---

## 7. Обеспечение удобства интерфейса

Особое внимание уделялось пользовательскому интерфейсу:

* удобная навигация;
* адаптивный дизайн;
* понятное расположение элементов;
* простота работы с публикациями.

---

## 8. Проверка корректности работы

В ходе тестирования были проверены:

* регистрация пользователей;
* корректность авторизации;
* создание и удаление постов;
* работа подписок;
* отображение скрытых публикаций;
* сортировка по тегам.

---

# 4. Выявленные ошибки в ходе выполнения задачи

В процессе разработки были выявлены следующие проблемы:

1. Ошибки при авторизации пользователей.
2. Некорректное отображение тегов.
3. Дублирование постов в ленте.
4. Ошибки удаления комментариев.
5. Проблемы отображения скрытых постов.
6. Отсутствие проверки пустых полей.
7. Ошибки при сортировке публикаций.
8. Низкая производительность при большом количестве постов.

---

# 5. Рекомендации по устранению ошибок

## 1. Усиление проверки данных

Необходимо реализовать дополнительную валидацию:

* проверка длины пароля;
* проверка корректности email;
* запрет пустых полей.

---

## 2. Использование хеширования паролей

Для повышения безопасности рекомендуется применять:

* bcrypt;
* SHA-256;
* двухфакторную аутентификацию.

---

## 3. Оптимизация запросов к базе данных

Следует:

* добавить индексы;
* оптимизировать SQL-запросы;
* использовать кэширование.

---

## 4. Проверка прав доступа

Рекомендуется дополнительно проверять:

* принадлежность поста пользователю;
* права на удаление комментариев;
* доступ к скрытым публикациям.

---

## 5. Улучшение системы тегов

Необходимо:

* исключить дублирование тегов;
* реализовать автоматические подсказки;
* добавить популярные категории.

---

## 6. Повышение производительности

Для ускорения работы системы рекомендуется:

* реализовать пагинацию;
* ограничить количество запросов;
* использовать асинхронную загрузку данных.

---

## 7. Улучшение интерфейса

Следует:

* сделать адаптацию под мобильные устройства;
* улучшить навигацию;
* добавить уведомления пользователям.

---

# 6. Итог

В результате выполнения кейс-задачи было разработано полноценное web-приложение для ведения блога с поддержкой регистрации пользователей, публикации постов, подписок, тегов и комментариев.

Система обеспечивает:

* удобное взаимодействие пользователей;
* управление контентом;
* персонализацию ленты;
* безопасность данных;
* возможность дальнейшего масштабирования проекта.

Разработанное приложение может использоваться как основа для создания полноценной социальной платформы или корпоративного блога.
