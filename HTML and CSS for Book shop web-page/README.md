# Анализ выполнения задачи по разработке web-версии книжного магазина

# 1. Анализ разработки пользовательского интерфейса

В ходе выполнения задачи был разработан пользовательский интерфейс web-приложения книжного магазина. Основной целью являлось создание удобной и понятной системы взаимодействия пользователя с библиотекой книг.

Интерфейс включает:

* главную страницу;
* систему авторизации;
* раздел просмотра книг;
* поиск и фильтрацию;
* функции покупки и аренды.

При разработке интерфейса особое внимание уделялось простоте навигации и адаптивности страницы.

---

# 2. Анализ реализации двух интерфейсов

В системе были реализованы два типа пользователей:

* пользователь;
* администратор.

Для каждого интерфейса предусмотрены отдельные функции и права доступа.

### Пользователь:

* просматривает книги;
* сортирует каталог;
* арендует и покупает книги.

### Администратор:

* добавляет новые книги;
* удаляет книги;
* изменяет стоимость;
* управляет доступностью.

Разделение ролей позволило реализовать базовую систему разграничения прав доступа.

---

# 3. Анализ базы данных книг

Для хранения информации о книгах использовалась структура объектов JavaScript.

Каждая книга содержит:

* название;
* автора;
* категорию;
* год написания;
* цену;
* статус доступности.

Созданная структура позволила удобно реализовать:

* поиск;
* сортировку;
* фильтрацию;
* изменение информации о книгах.

---

# 4. Анализ системы сортировки и поиска

В приложении реализована фильтрация по:

* автору;
* категории;
* году написания.

Система поиска позволяет быстро находить нужные книги в библиотеке. Реализация фильтрации значительно улучшила удобство использования системы при большом количестве данных.

---

# 5. Анализ системы аренды и покупки

Одной из основных функций приложения стала возможность покупки и аренды книг.

Были реализованы:

* покупка книги;
* аренда на 2 недели;
* аренда на 1 месяц;
* аренда на 3 месяца.

Также была добавлена функция автоматического напоминания пользователю об окончании срока аренды.

---

# 6. Анализ административной панели

Административная панель предоставляет возможность управления библиотекой.

Администратор может:

* добавлять новые книги;
* удалять существующие;
* изменять статус доступности;
* управлять стоимостью.

Данный функционал позволяет поддерживать актуальность каталога.

---

# 7. Анализ безопасности и проверки данных

В ходе разработки были реализованы базовые проверки:

* контроль прав администратора;
* проверка доступности книг;
* ограничение некоторых действий для обычных пользователей.

Однако система требует дополнительной защиты данных и более надежной авторизации.

---

# 8. Анализ удобства использования

Интерфейс приложения был разработан с учетом удобства пользователя:

* простая структура страниц;
* логичное расположение элементов;
* минимальное количество действий для выполнения операций.

Это позволило сделать приложение понятным даже для неподготовленных пользователей.

---

# 9. Анализ производительности системы

При небольшом количестве книг приложение работает стабильно и быстро. Однако при увеличении объема данных возможно снижение производительности из-за хранения информации в массивах JavaScript без использования полноценной базы данных.

---

# Выявленные ошибки в ходе выполнения задачи

В процессе разработки были выявлены следующие проблемы:

1. Отсутствие полноценной системы регистрации и хранения пользователей.
2. Отсутствие подключения к реальной базе данных.
3. Недостаточная защита пользовательских данных.
4. Возможность изменения данных через консоль браузера.
5. Отсутствие серверной части приложения.
6. Уведомления об аренде реализованы только в рамках открытой страницы.
7. Отсутствие сохранения данных после обновления страницы.
8. Нет проверки корректности введенных данных.
9. Возможны ошибки при добавлении пустых значений.
10. Отсутствует система онлайн-оплаты.

---

# Рекомендации по устранению выявленных ошибок

## 1. Подключение полноценной базы данных

Рекомендуется использовать:

* MySQL;
* PostgreSQL;
* MongoDB.

Это позволит:

* сохранять данные;
* ускорить работу системы;
* обеспечить надежное хранение информации.

---

## 2. Создание серверной части

Необходимо реализовать backend с использованием:

* Node.js;
* Express;
* PHP;
* Django.

Серверная часть обеспечит:

* безопасную обработку данных;
* авторизацию;
* хранение информации.

---

## 3. Улучшение системы авторизации

Следует:

* добавить регистрацию пользователей;
* использовать хеширование паролей;
* реализовать JWT или сессии;
* ограничить доступ к административной панели.

---

## 4. Усиление проверки данных

Необходимо реализовать:

* проверку пустых полей;
* ограничение длины текста;
* проверку форматов данных;
* обработку ошибок.

---

## 5. Реализация постоянного хранения данных

Следует сохранять:

* книги;
* пользователей;
* аренды;
* покупки.

Это позволит избежать потери информации после перезагрузки страницы.

---

## 6. Реализация реальных уведомлений

Для напоминаний рекомендуется использовать:

* email-уведомления;
* push-уведомления;
* Telegram-бота.

Это позволит автоматически уведомлять пользователей об окончании аренды.

---

## 7. Повышение безопасности системы

Рекомендуется:

* ограничить доступ к административным функциям;
* использовать HTTPS;
* защитить API;
* реализовать проверку токенов доступа.

---

## 8. Оптимизация производительности

При увеличении количества книг рекомендуется:

* использовать пагинацию;
* оптимизировать запросы;
* внедрить кэширование;
* загружать данные частями.

---

# Итог

В результате выполнения задачи была разработана web-версия книжного магазина с пользовательским и административным интерфейсом.

Система поддерживает:

* просмотр каталога;
* поиск и сортировку;
* покупку и аренду книг;
* управление библиотекой;
* изменение статусов и цен;
* автоматические напоминания.

Разработанное приложение может быть использовано как основа для создания полноценной системы электронного книжного магазина.
